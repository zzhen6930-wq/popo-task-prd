WrokHub Studio · AI 工厂「知识」

复用成熟的「知识本」能力，让用户把多来源资料统一沉淀、自动整理成 Wiki，并支持 LLM 语义检索，补齐平台知识检索能力。
📅 2026-07-23 🏷️ v1.0 👤 作者：青远（张振） 🧩 归属：WrokHub Studio / AI 工厂 · 知识
1背景与目标

当前 WrokHub Studio 平台没有知识库能力，Agent 回答只能依赖模型自身知识与实时工具返回，缺少一块「用户提供的、可沉淀、可检索」的背景资料底座，导致：

  • 无处沉淀：用户想让 Agent 参考自己的文档 / 资料，没有统一入口挂载
  • 检索缺失：多个来源的资料无法统一语义检索，Agent 拿不到相关背景
  • 整理靠人：一堆零散文件缺少结构化组织，用不起来

本期目标：复用平台已有的「知识本」能力补齐知识检索。用户在「知识」菜单下以「知识本」为对象上传多来源资料（知识本 / 云空间 / 本地文件），这些资料统一落到一个知识本并分配唯一知识本 ID，从而直接复用知识本现有的知识整理、Wiki 生成、LLM 语义检索能力。绑定 Agent 在「Agent 管理」完成。

🔑 一句话定位
「知识」不是重造轮子，而是给「知识本」换一层面向 Agent 的外衣：用户上传资料 = 系统给他建/复用一个知识本，多来源文件汇入同一个知识本，天然获得统一检索 + Wiki。

3能力构建原理（复用知识本）

核心思路：不新建独立知识库系统，而是复用知识本。用户通过三种方式添加资料时，无论来源如何，最终都被汇入同一个知识本，继续享受知识本现有的整理 / Wiki / 检索能力。

5添加资料的三种方式

点击「+ 添加资料」后，提供三种添加入口，添加结果统一汇入当前 Agent 的知识本：

📚 1. 选择知识本
绑定有权限的个人 / 团队知识本
☁️ 2. 选择云空间文件
从 POPO 云空间选文档 / 文件夹
⬆️ 3. 上传本地文件
上传 PDF / Word / PPT 等本地文件

5.1 选择知识本

  • 列出用户有权限的知识本，可同时选择多个
  • 每个知识本显示「来源数量、创建人」（如「19 个来源 · 创建时间…」）
  • 添加后，把选中知识本内全部来源（文件）添加到当前列表——选知识本即选它里面的文件
  • 底部显示「已选择 N 个」，点击「添加」完成汇入

5.2 选择云空间文件

  • 支持关键字、链接搜索云空间文档
  • 支持多选文档添加
  • 支持选择云空间里的文件夹，将文件夹下的文档批量加进来
  • 底部显示「添加(N)」，N 为最终纳入的文档数

5.3 上传本地文件

  • 打开系统上传组件，正常上传 PDF / Word / PPT 等本地文件
  • 上传即写入知识本，作为一个知识来源
6Wiki 自动生成与编辑
  • 资料上传后自动生成 Wiki 视图（知识本现有能力，无需额外配置）
  • Wiki 由一个默认 Prompt 负责整理目录结构（对来源做归纳分组）
  • 用户可自行编辑该 Prompt / 结构：例如「请帮我重新组织这个知识库的 Wiki 结构，把资源自动归类到以下分组：服务端、前端、策划」，点击「重新生成视图」按新结构重排
复用说明
Wiki 生成、默认 Prompt、自定义重排均为知识本现有能力，本期直接复用，不重复建设。

8验收标准
  1. 「知识」列表页按用户 + Agent维度展示，进入后可见该 Agent 全部知识。
  2. 详情页可切换资料 / Wiki 两个视图。
  3. 三种添加方式均可用，且三种来源的文件最终写入同一个知识本。
  4. 选知识本可多选，显示来源数量 + 创建人，添加后按来源数量把文件加入列表。
  5. 云空间支持关键字 / 链接搜索、多选文档、文件夹批量添加。
  6. 本地文件正常上传入库。
  7. 上传后自动生成 Wiki 视图，且默认 Prompt 可编辑并重新生成。
  8. 资料支持查看 / 删除，不支持编辑。
  9. 知识本内全部来源可被 LLM 统一语义检索。
