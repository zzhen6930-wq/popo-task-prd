Fabric · AI 工厂「记忆」能力（一期）

为 Agent 补齐分层记忆 + 异步做梦能力，让智能体像数字同事一样越用越懂你
📅 2026-07-22 🏷️ v1.0 👤 作者：青远（张振） 🧩 归属：Fabric / AI 工厂
1背景与目标

Fabric 是面向组织 / 用户的 Agent 配置平台。当前 Agent 没有记忆能力——每次对话都是"失忆"状态，导致：

  • 用户反复交代同样的背景、偏好、身份，Agent 记不住，体验断裂
  • 用户纠正过的错误、明确过的禁忌，下次对话又犯，无法沉淀"教训"
  • 跨会话的项目上下文、决策、事实无法积累，Agent 无法越用越聪明
  • 只能靠把信息硬塞进 Prompt / Skill，规模一大就超上下文、命中率低

本期目标：为 Fabric 补齐一套分层记忆架构（基础文件 + 用户记忆）+ 异步做梦（Auto-Dream）机制，并在「AI 工厂」下新增「记忆」菜单支持后台可视化管理。核心机制对标 Claude Tag（分层文件记忆 + 异步做梦 + 按需注入），短期记忆参考 OpenClaw 的会话缓冲设计。

🔑 一句话定位
让 Agent 从「群聊摘要机器人」升级为「具备睡眠整理能力的数字同事」：对话时保持实时流畅，空闲时在"后台做梦"消化信息，沉淀为可复用的长期知识。
2核心概念与术语
术语说明
记忆（Memory）Agent 持久化存储的信息，以文件为载体（Markdown 结构化文本）
基础文件相对静态的底座文件，含 soul（智能体人格设定）与 user（用户身份档案）两个文件
用户记忆随交互动态积累的记忆，含短期记忆与长期记忆
短期记忆当前 / 近期会话内的临时缓冲（原始对话日志），未经"做梦"提炼（参考 OpenClaw）
长期记忆跨会话持久化的知识，由多个分类文件组成（参考 Claude Tag 分层）
异步做梦（Auto-Dream）空闲时段后台守护进程，把短期记忆批量提炼、压缩、去重，写入长期记忆
按需注入对话时只注入相关记忆片段，而非全量背诵；大文档只存索引、用时再读
3记忆分层架构（总览）

记忆分为 两大分类：基础文件 与 用户记忆。用户记忆再分短期与长期，长期记忆由多个分类文件组成。

📁 基础文件（相对静态）
soul 人格
智能体的角色设定、语气风格、行为准则、能力边界。定义"我是谁、我怎么说话"。
user 身份
用户的身份档案：姓名 / 角色 / 部门 / 基本背景等相对稳定的画像。定义"你是谁"。
🧠 用户记忆（动态积累）
短期记忆 会话缓冲
近期会话原始日志，做梦前的临时缓存。滚动窗口，有容量 / 时效上限。
长期记忆 多分类文件
由多个分类文件组成（见第 5 节）：偏好记忆 / 事实记忆 / 反馈记忆 / 引用记忆。
💡 为什么 user（基础文件）与偏好记忆（长期记忆）要分开？
user 文件 = 相对稳定的身份档案（姓名 / 角色 / 部门），变动少、可人工维护为主；偏好记忆 = 交互中动态学到的习惯与风格，由做梦持续增量更新。二者读写节奏与来源不同，故分层管理。
4基础文件详解

4.1 soul 文件（智能体人格）

用户场景：配置者希望 Agent 始终保持一致的角色和语气（如"资深法务顾问，严谨、引用条款"），不因对话漂移。

解决问题：人格 / 行为准则散落在 Prompt 里易被覆盖、难维护；沉淀为独立文件后稳定、可复用、可版本管理。

机制：soul 文件在每次对话必注入系统上下文最前部；由配置者维护，做梦机制不改写 soul（人格稳定）。

4.2 user 文件（用户身份档案）

用户场景：用户不想每次重复"我是财务部的张三，负责个税申报"。

解决问题：身份类信息高频复用、相对稳定，硬塞 Prompt 浪费上下文。

机制：user 文件在对话时必注入；支持用户 / 管理员人工编辑，做梦机制可对身份类新事实做增量补充（如识别到新角色），但以人工维护为准。

5用户记忆详解

5.1 短期记忆（参考 OpenClaw）

用户场景：同一次 / 近几次对话里，用户说"就用刚才那个方案"，Agent 要能记得"刚才"。

解决问题：单纯依赖模型上下文窗口，长对话会溢出、跨会话即丢失。

机制：

  • 会话缓冲：以滚动窗口保存近期原始对话日志（含关键消息、决策片段）
  • 容量 / 时效上限：超出上限的旧内容进入"待做梦"队列，等待异步提炼
  • 低信噪比过滤：忽略表情、闲聊等无价值内容，只留有信息量的片段
  • 做梦后清理：已被提炼进长期记忆的短期内容按策略清理，避免无限膨胀

5.2 长期记忆（多分类文件，参考 Claude Tag）

用户场景：Agent 要记住"用户偏好简洁回答""项目用 PostgreSQL 15，升级前须跑 X 脚本""上次因为漏了权限校验踩过坑"。

解决问题：把碎片信息沉淀为分类清晰、可复用的知识，而非一坨扁平摘要。

机制：长期记忆由多个分类文件组成，各文件更新节奏不同：

长期记忆文件内容来源作用更新频率
偏好记忆
Preference		用户的习惯、风格、纠正表达记住"你喜欢什么风格 / 怎么沟通"每次交互后增量更新
事实记忆
Fact / Project		项目决策、技术约定、里程碑、稳定事实记住"我们在做什么、怎么做才对"异步做梦整合
反馈记忆
Feedback		用户的否定、修正、边界与禁忌记住"什么不能做、上次错在哪"强制即时记录（不等做梦）
引用记忆
Reference		外部文档链接、API 规范、知识库索引知道"去哪里找详细信息"按需添加
💡 引用而非背诵
对长文档 / 大段讨论，长期记忆只存索引 / 链接到「引用记忆」，需要时再读取原文（可对接知识本 RAG），而非把原文总结硬塞进核心记忆——控制记忆体积、提升命中率。
⚠️ 反馈记忆为什么"强制即时"
禁忌 / 纠正类信息若等到做梦才记，可能在下一轮就重复犯错。因此反馈记忆在识别到的当下即时写入，不进"待做梦"队列。
6异步做梦机制（Auto-Dream · 核心）

用户场景：用户白天和 Agent 聊了很多，晚上 / 空闲时 Agent 自动"消化"，第二天变得更懂用户。

解决问题：实时总结会占用推理资源、打断对话流畅度；把提炼放到空闲异步做，兼顾实时性与知识沉淀。

6.1 触发时机

  • 空闲触发：会话结束后静默 N 分钟无新活动
  • 定时触发：每日固定时段（类比人类睡眠 REM）批量处理当日短期记忆
  • 阈值触发：短期记忆缓冲达到容量上限时提前触发

6.2 做梦流程

短期记忆
  • 压缩与去重：碎片对话压缩成高密度结构化笔记（Markdown），剔除冗余，抽取可复用的规则 / 决策 / 事实
  • 分类归档：提炼结果按语义归入偏好 / 事实 / 反馈 / 引用对应文件
  • 合并策略：同类信息增量合并而非覆盖；与已有条目冲突时保留更新的、标注时间戳
  • 做梦日志：每次做梦产出一条 Dream Log（处理了哪些、沉淀了什么），后台可查、可追溯

6.3 与人工编辑的防冲突

  • 用户 / 管理员在后台人工编辑过的记忆条目优先级更高，做梦不静默覆盖人工内容
  • 做梦对人工条目只做补充 / 建议，冲突项标记待确认，避免"越改越乱"
🔑 知识内化闭环（对标 Claude Tag）
感知 → 暂存（短期记忆）→ 触发（空闲 / @ / 关键事件）→ 做梦（提炼规则）→ 固化（写长期记忆）→ 应用（下次直接调用，无需重读历史）。
7记忆的读取与注入（按需注入）

对话发生时，按以下优先级组装上下文，只注入相关片段，不全量背诵：

记忆注入策略
soul必注入，置于系统提示最前
user必注入
短期记忆注入近期窗口内的会话缓冲
长期记忆（偏好/事实/反馈）按当前问题相关性检索注入 Top-K 片段；反馈记忆（禁忌）优先注入
引用记忆默认只注入索引；命中需要时再拉取原文（可走知识本 RAG）

8.1 列表页（以 Agent 为维度）

列表展示当前空间下的 Agent，点击进入记忆详情。列字段：Agent 名称 / 记忆文件数 / 短期记忆量 / 上次做梦时间 / 操作。

8.2 详情页（横向 Tab 切换两大分类）

进入某 Agent 后，用横向 Tab 切换查看 / 管理两类记忆：

  • Tab ①「基础文件」：soul、user 两个文件，支持查看 / 编辑
  • Tab ②「用户记忆」：短期记忆（会话缓冲预览 + 清理）+ 长期记忆（偏好 / 事实 / 反馈 / 引用 多个分类文件，支持查看 / 编辑 / 删除）
💡 管理能力（一期）
各记忆文件支持查看、编辑、删除；可查看 Dream Log（做梦记录）；可手动触发一次做梦。人工编辑与做梦按第 6.3 节防冲突策略协作。
9典型场景串联
场景 1 · 偏好沉淀
用户多次要求"回答简洁点"
短期记忆记录这几次纠正 → 空闲做梦提炼出"用户偏好简洁、要点式回答" → 写入偏好记忆 → 下次对话自动注入，无需再交代。
场景 2 · 禁忌即时记
用户说"以后别用外部 API 示例，我们内网不通"
识别为反馈 / 禁忌 → 即时写入反馈记忆（不等做梦）→ 下一轮起优先注入，避免重复犯错。
场景 3 · 引用而非背诵
用户丢来一篇 50 页技术规范
长期记忆只在引用记忆存"规范链接 + 摘要索引" → 用户问到相关问题时按需拉取原文（对接知识本 RAG），不把全文塞进记忆。
场景 4 · 后台治理
管理员发现一条记忆记错了
进入「记忆」菜单 → 选中 Agent → 用户记忆 Tab → 编辑 / 删除对应长期记忆条目；该人工修改优先于后续做梦，不被静默覆盖。
10一期范围

✅ 一期要做

  • 分层记忆架构：基础文件（soul / user）+ 用户记忆（短期 / 长期）
  • 长期记忆四类分类文件：偏好 / 事实 / 反馈 / 引用
  • 异步做梦：空闲 / 定时 / 阈值触发 + 提炼归档 + Dream Log
  • 反馈记忆强制即时记录
  • 按需注入（相关性检索 + 引用只存索引）
  • 「记忆」菜单：Agent 列表 + 详情横向 Tab（基础文件 / 用户记忆）
  • 记忆文件的查看 / 编辑 / 删除 + 手动触发做梦
  • 人工编辑 vs 做梦的防冲突策略

❌ 一期不做

  • 记忆的版本回溯 / 时间线 diff（二期）
  • 跨 Agent / 跨用户的记忆共享与继承
  • 记忆条目级的细粒度权限与审批流
  • 做梦策略的可视化编排（自定义提炼规则）
  • 记忆容量计费与配额后台
  • 多模态记忆（图片 / 音频记忆）
11权限与依赖
  • 菜单可见性：「记忆」菜单对具备该空间权限的配置者可见；记忆数据按 Agent + 用户维度隔离
  • 对模型的依赖：做梦的提炼、注入的相关性检索依赖 LLM + 向量检索能力
  • 对调度的依赖：定时 / 空闲做梦依赖调度能力（可复用「自动化任务」调度引擎）
  • 对知识本的依赖：引用记忆的原文拉取可对接知识本 RAG（第 7 节）
  • 数据安全：记忆含用户敏感信息，须加密存储、访问审计、支持一键清除
12交付与验收
  • 「记忆」菜单出现在 AI 工厂下，Agent 列表 → 详情横向 Tab 可正常切换查看基础文件 / 用户记忆
  • 基础文件（soul / user）在对话中必注入并生效
  • 短期记忆按窗口滚动，超限内容进入待做梦队列
  • 异步做梦可被空闲 / 定时 / 阈值触发，产出 Dream Log，长期记忆被正确分类写入
  • 反馈 / 禁忌类可即时写入，下一轮对话即生效不再犯
  • 引用记忆只存索引，按需拉取原文
  • 记忆文件可查看 / 编辑 / 删除；人工编辑不被做梦静默覆盖
  • 场景 1~4 可端到端跑通
13二期方向（不在一期范围）
  • 记忆版本回溯与时间线 diff、一键回滚
  • 跨 Agent / 团队级记忆共享与继承
  • 做梦提炼规则可视化编排
  • 记忆条目级权限、审批与合规流程
  • 多模态记忆
  • 记忆容量配额与计费
