WorkHub Studio · AI 工厂「MCP」

补齐平台链接外部系统的能力：内置一批常用 MCP，并支持自助添加，把可用的 MCP 服务统一管理起来供 Agent 调用。
📅 2026-07-24 🏷️ v1.0 👤 作者：青远（张振） 🧩 归属：WorkHub Studio / AI 工厂 · MCP
1背景与目标

当前 WorkHub Studio 平台缺少通过 MCP（Model Context Protocol）链接外部系统的能力，Agent 无法便捷地接入企业内外部工具与数据源。本期补齐该能力：

  • 内置一批常用 MCP：开箱即用，覆盖 POPO 生态、研发、法务、企业信息等高频场景
  • 支持自助添加 MCP：支持 STDIO / SSE / HTTP 三种传输类型，可上传自定义图标
  • 统一管理：本模块只负责把「可用的 MCP」集中管理与呈现，供 Agent 在配置时选用
🔑 一句话定位
MCP 模块是平台的「外部系统接口货架」：把内置 + 自定义的 MCP 服务统一陈列管理；Agent 与 MCP 的绑定 / 启用不在本模块，本模块只解决「有哪些可用」。
2核心概念与术语
术语说明
MCPModel Context Protocol，标准化协议，用于让 Agent / 模型链接外部系统的工具与数据
内置 MCP平台预置、开箱即用的 MCP 服务，以卡片形式陈列
自定义 MCP用户通过「添加 MCP」自助接入的服务，需配置传输类型与连接参数
传输类型MCP 服务的连接方式：STDIO（本地进程） / SSE（服务端推送） / HTTP（HTTP 流）

4MCP 页面（卡片陈列）

在「AI 工厂」下新增「MCP」菜单。页面以卡片形式陈列所有可用 MCP（非列表），卡片较小、横向紧凑；页面右上角为「＋ 添加 MCP」入口。

4.1 卡片内容

每张 MCP 卡片仅展示三项：

元素说明
图标MCP 的图标（内置自带；自定义可选默认或上传）
名称MCP 名称，单行超出省略
描述一句话说明该 MCP 的用途，最多两行超出省略
卡片不展示「添加/启用」按钮
卡片上不放「添加」按钮，也不做任何绑定/启用声明。本模块作用是把可用的 MCP 管理起来；Agent 选用 MCP 在 Agent 配置侧进行，不在本页。
5添加 MCP（弹窗配置）

点击右上角「＋ 添加 MCP」弹出「MCP 配置」弹窗。公共字段 + 随传输类型动态变化的字段：

5.1 公共字段

字段说明
服务器名称MCP 名称，如 My Custom Server
传输类型下拉选择 STDIO / SSE / HTTP，切换后下方字段联动变化
图标提供默认图标可选，也支持手动上传（PNG / JPG，≤ 1 MB，推荐 256×256）
备注（可选）提供 MCP 文档或说明，告知平台如何及何时使用此 MCP

5.2 传输类型动态字段

传输类型专属字段
STDIO命令（如 python / node / ./script.sh）、参数（可增删多条）、环境变量（可选，键值对，可增删多条）
SSE服务器 URL（如 https://mcp.yourserver.com/sse）、自定义 headers（可选，键值对，可增删多条）
HTTP服务器 URL（如 https://mcp.yourserver.com/mcp）、自定义 headers（可选，键值对，可增删多条）
保存
底部「保存」提交；「保存」旁的下拉可「保存并添加下一个」。保存成功后新 MCP 以卡片形式加入陈列。
6边界与非目标

✅ 本期做

  • 内置 11 个常用 MCP，卡片陈列（小卡片）
  • 右上角添加 MCP，支持 STDIO / SSE / HTTP
  • 默认图标 + 手动上传
  • 卡片展示 图标 / 名称 / 描述

🚫 本期不做

  • 不在本页做 Agent↔MCP 绑定 / 启用（在 Agent 配置侧）
  • 卡片不放添加/启用按钮、不做声明
  • 不做 MCP 连通性健康检查 / 监控（后续）
  • 不做 MCP 工具级权限精细化（后续）
7验收标准
  1. 「MCP」菜单以小卡片陈列全部可用 MCP，非列表；右上角有「＋ 添加 MCP」。
  2. 内置 11 个 MCP 正确展示：POPO 云空间 / 日程 / 任务、易协作、KM、网易邮箱、GitHub、Figma、CodeMaker、北大法宝、天眼查。
  3. 每张卡片仅展示 图标 / 名称 / 描述，无添加/启用按钮、无声明。
  4. 添加 MCP 弹窗含公共字段（名称 / 传输类型 / 图标 / 备注），图标支持默认选择 + 上传。
  5. 传输类型可选 STDIO / SSE / HTTP，切换后动态字段联动：STDIO 出命令/参数/环境变量；SSE、HTTP 出服务器 URL/自定义 headers。
  6. 参数 / 环境变量 / headers 支持增删多条。
  7. 保存后新 MCP 以卡片加入陈列；支持「保存并添加下一个」。
