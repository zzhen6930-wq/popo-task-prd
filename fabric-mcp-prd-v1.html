WorkHub Studio · AI 工厂「MCP」

补齐平台链接外部系统的能力：内置一批常用 MCP，并支持自助添加，把可用的 MCP 服务统一管理起来供 Agent 调用。
📅 2026-07-24 🏷️ v1.0 👤 作者：青远（张振） 🧩 归属：WorkHub Studio / AI 工厂 · MCP
1背景与目标

当前 WorkHub Studio 平台缺少通过 MCP（Model Context Protocol）链接外部系统的能力，Agent 无法便捷地接入企业内外部工具与数据源。本期补齐该能力：

  • 内置一批常用 MCP：开箱即用，覆盖 POPO 生态、研发、法务、企业信息等高频场景
  • 支持自助添加 MCP：支持 STDIO / SSE / HTTP 三种传输类型，可上传自定义图标
  • 统一管理：本模块只负责把「可用的 MCP」集中管理与呈现，供 Agent 在配置时选用
🔑 一句话定位
MCP 模块是平台的「外部系统接口货架」：把内置 + 自定义的 MCP 服务统一陈列管理；Agent 与 MCP 的绑定 / 启用不在本模块，本模块只解决「有哪些可用」。
2核心概念与术语
术语说明
MCPModel Context Protocol，标准化协议，用于让 Agent / 模型链接外部系统的工具与数据
内置 MCP平台预置、开箱即用的 MCP 服务，以卡片形式陈列
自定义 MCP用户通过「添加 MCP」自助接入的服务，需配置传输类型与连接参数
传输类型MCP 服务的连接方式：STDIO（本地进程） / SSE（服务端推送） / HTTP（HTTP 流）

4MCP 页面（卡片陈列）

在「AI 工厂」下新增「MCP」菜单。页面以卡片形式陈列所有可用 MCP（非列表），卡片较小、横向紧凑；页面右上角为「＋ 添加 MCP」入口。

4.1 卡片内容

每张 MCP 卡片仅展示三项：

元素说明
图标MCP 的图标（内置自带；自定义默认给一个占位图标，支持手动上传替换）
名称MCP 名称，单行超出省略
描述一句话说明该 MCP 的用途，最多两行超出省略
卡片不展示「添加/启用」按钮
卡片上不放「添加」按钮，也不做任何绑定/启用声明。本模块作用是把可用的 MCP 管理起来；Agent 选用 MCP 在 Agent 配置侧进行，不在本页。
内置 MCP 只读
内置 MCP 由平台预置提供给租户，仅供陈列与选用，不可编辑、不可删除。租户可通过右上角「＋ 添加 MCP」新增自定义 MCP。
5添加 MCP（弹窗配置）

点击右上角「＋ 添加 MCP」弹出「MCP 配置」弹窗。弹窗按当前租户类型分叉（顶部可切换租户），并遵循用户最小化输入原则——除必须由用户/租户提供的参数外，其余能由平台预填的都由平台预填。

5.1 公共字段（两类租户通用）

字段说明
服务器名称MCP 名称，如 My Custom Server
图标默认给一个占位图标（默认选中），支持手动上传替换（PNG / JPG，≤ 1 MB，推荐 256×256）；不提供图标候选排
备注（可选）提供 MCP 文档或说明，告知平台如何及何时使用此 MCP

5.2 AgentHub 租户（C 端）· 只选鉴权方式

C 端不暴露传输类型与连接参数（由平台预填），仅需选择鉴权方式，凭据强制绑定用户维度：

鉴权方式说明
OAuth 授权后台仅填应用级信息（如 client_id）；用户点「连接」跳授权页各自授权，token 存用户维度。适用 GitHub / Figma / 腾讯文档等有 OAuth 的服务
用户输入 API Key后台声明「需 API Key」；用户在前台各自填写自己的 Key（存用户维度）。适用天眼查 / 北大法宝等只发 Key 的服务
静默授权针对公司内部应用（POPO 云空间 / 日程 / 任务、KM、易协作等），复用登录态免密、用户无感

5.3 其他租户（B 端）· 自填传输参数（统一凭据）

B 端不展示「鉴权方式」字段，由租户自行填写 STDIO / SSE / HTTP 三种传输方式的参数——这即是租户统一提供凭据，绑定租户维度，供本部门员工无感调用（平台按员工身份记调用审计）：

传输类型专属字段
STDIO命令（如 python / node / ./script.sh）、参数（可增删多条）、环境变量（可选，键值对，可增删多条）
⚠️ STDIO 靠本地进程运行，SaaS 可用性待研发评估，本期先保留字段
SSE服务器 URL（如 https://mcp.yourserver.com/sse）、自定义 headers（可选，键值对，可增删多条）
HTTP服务器 URL（如 https://mcp.yourserver.com/mcp）、自定义 headers（可选，键值对，可增删多条）
保存
底部「保存」提交；「保存」旁的下拉可「保存并添加下一个」。保存成功后新 MCP 以卡片形式加入陈列。
6MCP 连接与鉴权逻辑

除交互外，MCP 作为「链接外部系统」的通道，需明确以下核心逻辑：

6.1 传输类型与连接建立

传输类型连接方式连接参数
STDIO平台在 Sandbox / 执行环境内本地拉起进程，通过标准输入输出通信命令 + 参数 + 环境变量
SSE平台以 Server-Sent Events 长连接对接远端 MCP 服务服务器 URL + 自定义 headers
HTTP平台以 HTTP（Streamable） 对接远端 MCP 服务服务器 URL + 自定义 headers

6.2 鉴权（Authentication）

核心原则：凭据的归属层级决定合规性。后台添加 MCP 时只声明「鉴权方式」，不直接填写具体凭据（如 header 值 / 环境变量值）——否则等于全体终端用户共用平台/管理员的一份账号，越权且不合规。凭据按下述场景分别绑定到用户维度或租户维度。

① 公司内部 MCP（POPO 云空间 / 日程 / 任务、KM、易协作等）

场景鉴权凭据归属
C 端 / B 端 通用静默授权（复用登录态、免密）无感，用户/员工无需配置

② 非公司内部 MCP · C 端场景（AgentHub 租户）——凭据强制绑定用户维度：

鉴权方式交互凭据归属
OAuth 授权（GitHub / Figma / 腾讯文档等有 OAuth 的服务）后台配 OAuth 应用信息；用户点「连接」跳授权页同意后回调拿 token用户维度，各人各自 token
用户输入 API Key（天眼查 / 北大法宝等只发 API Key 的服务）后台声明「需 API Key」；用户在前台填写自己的 Key 后使用用户维度，各人各自 Key

③ 非公司内部 MCP · B 端场景（B 端自建租户，给本部门员工使用）——凭据绑定租户维度：

鉴权方式交互凭据归属 / 审计
租户统一凭据B 端管理员配置一份统一凭据（统一账号的 OAuth token / API Key），员工无感调用租户维度共享；平台按员工身份记录调用审计日志（谁、何时、调了哪个工具），满足审计合规
两类租户弹窗分叉
C 端（AgentHub）：只选鉴权方式（OAuth 授权 / 用户输入 API Key / 静默授权），不暴露传输参数，凭据绑用户维度、强制各自授权。其他租户（B 端）：不展示鉴权方式，自填 STDIO / SSE / HTTP 传输参数（即租户统一凭据），绑租户维度，不支持员工各自授权。
🔑 凭据安全
所有凭据（OAuth token / API Key / header 值 / 环境变量值）加密存储，界面脱敏展示、不回显明文；删除 MCP 或用户解绑时一并清除对应凭据。凭据仅在运行时解密用于连接，不写入日志、不下发前端。传输层承载：STDIO 经环境变量注入本地进程，SSE / HTTP 经请求头（如 Authorization: Bearer <token>）携带——但承载的具体凭据值来自上述用户/租户维度的凭据库，而非 MCP 定义本身。

6.3 生效范围与调用链路

  • 租户隔离：MCP（含自定义与凭据）归属当前租户，仅本租户 Agent 可选用，跨租户不可见。
  • 调用链路：Agent 配置侧勾选可用 MCP → 运行时平台按传输类型建立连接，并按 C 端（用户维度）/ B 端（租户维度）取对应凭据完成鉴权 → 拉取 MCP 暴露的工具（tools）/ 资源（resources）供模型调用。
  • 连通性：保存时可做一次连接自检（可选），失败给出错误原因；持续健康监控本期不做（见第 7 节）。

6.4 异常与降级

  • 连接失败 / 鉴权失效：调用侧返回明确错误（超时 / 401 / 命令不存在等），不阻断 Agent 其它能力。
  • 凭据过期 / 失效：C 端提示用户重新授权 / 更新自己的 Key；B 端提示管理员更新租户统一凭据。
7边界与非目标

✅ 本期做

  • 内置 11 个常用 MCP，卡片陈列（小卡片）
  • 右上角添加 MCP，支持 STDIO / SSE / HTTP
  • MCP 管理：新增自定义 MCP；自定义 MCP 支持编辑 / 删除，内置 MCP 只读
  • 单默认图标 + 手动上传替换
  • 添加 MCP 弹窗按租户类型分叉（顶部切换租户），用户最小化输入
  • C 端（AgentHub）：只选鉴权方式——OAuth 授权 / 用户输入 API Key / 静默授权（内部应用），不暴露传输参数，凭据绑用户维度
  • 其他租户（B 端）：不展示鉴权方式，自填 STDIO / SSE / HTTP 参数（=租户统一凭据，绑租户维度 + 员工调用审计）；STDIO 的 SaaS 可用性待研发评估
  • 凭据加密存储、脱敏不回显
  • 卡片展示 图标 / 名称 / 描述

🚫 本期不做

  • 不在本页做 Agent↔MCP 绑定 / 启用（在 Agent 配置侧）
  • 卡片不放添加/启用按钮、不做声明
  • 不做 MCP 连通性健康检查 / 监控（后续）
  • 不做 MCP 工具级权限精细化（后续）
8验收标准
  1. 「MCP」菜单以小卡片陈列全部可用 MCP，非列表；右上角有「＋ 添加 MCP」。
  2. 内置 11 个 MCP 正确展示：POPO 云空间 / 日程 / 任务、易协作、KM、网易邮箱、GitHub、Figma、CodeMaker、北大法宝、天眼查。
  3. 每张卡片仅展示 图标 / 名称 / 描述，无添加/启用按钮、无声明。
  4. 添加 MCP 弹窗按当前租户类型分叉（顶部可切换），公共字段（名称 / 图标 / 备注）通用，图标支持默认选择 + 上传。
  5. C 端（AgentHub）弹窗只出「鉴权方式」下拉：OAuth 授权 / 用户输入 API Key / 静默授权，不暴露传输类型与连接参数；OAuth 仅填应用级 client_id。
  6. 其他租户（B 端）弹窗不展示鉴权方式，出「传输类型」STDIO / SSE / HTTP，切换后动态字段联动：STDIO 出命令/参数/环境变量；SSE、HTTP 出服务器 URL/自定义 headers；参数 / 环境变量 / headers 支持增删多条。
  7. 保存后新 MCP 以卡片加入陈列；支持「保存并添加下一个」。
  8. 内置 MCP 只读：不出现编辑 / 删除入口；自定义 MCP 卡片悬浮出现「✎ 编辑 / 🗑 删除」，编辑复用弹窗回填、删除二次确认。
  9. 图标默认给单个占位图标并选中，支持上传替换，无候选图标排。
  10. 后台添加 MCP 只声明鉴权方式、不直填终端用户凭据（不再直接填 Authorization header / 环境变量密钥值）。
  11. 鉴权凭据按维度绑定：C 端非内部工具由用户各自 OAuth 授权 / 各自输入 API Key（用户维度）；B 端非内部工具由租户自填传输参数即统一凭据（租户维度）且按员工身份记录调用审计；内部工具（POPO/KM/易协作等）走静默授权免密。B 端不支持员工各自授权。
  12. 所有凭据加密存储、脱敏不回显，删除 MCP / 用户解绑时清除对应凭据，不写日志、不下发前端。
  13. MCP 归属当前租户隔离；运行时按传输类型建连、按 C/B 维度取凭据鉴权，暴露工具供模型调用。
