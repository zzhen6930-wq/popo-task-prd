Fabric · AI 工厂「会话管理」升级

把会话从"只留首尾"升级为"全链路可观测"：结构化事件流 + Token / 耗时 / 时间，让每一次对话都能被排查、被度量。
📅 2026-07-23 🏷️ v1.0 👤 作者：青远（张振） 🧩 归属：Fabric / AI 工厂 · 会话
1背景与目标

Fabric 后台已有「会话」模块，可按 Bot / 用户 / 时间查询会话列表与详情。但当前实现只记录了用户输入与 Agent 最终输出，中间的模型思考、工具调用、知识检索、报错等过程信息全部缺失，导致：

  • 无法排查问题：Agent 回答不对 / 变慢 / 报错时，看不到中间发生了什么，只能靠猜
  • 无法度量成本与性能：单次会话消耗多少 Token、每一步耗时多久，都无从统计
  • 无法定位瓶颈：哪一步工具慢、哪次检索没命中、哪里报错重试，没有抓手
  • 渠道来源不清：无法区分会话来自 workflow 还是智能体机器人，难做分渠道分析

本期目标：升级会话管理的数据采集与可视化——将模型思考、调用工具、Skills、MCP、知识检索、模型输出、系统信息、报错信息等过程事件结构化记录并回放，并在每个事件上标注消耗 Token、耗时、开始时间；列表页补齐渠道类型与消耗 Token。参考同类 Agent 观测台（Trace / Timeline）的事件流交互形态（见参考图 1/2/3）。

🔑 一句话定位
给会话装上「行车记录仪 + 仪表盘」：完整回放对话每一步的过程，并量化每一步的 Token 与耗时，让排查从"靠猜"变成"看轨迹"。
2核心概念与术语
术语说明
会话（Session / Conversation）一个用户与某个 Bot / Agent 的完整对话，拥有唯一 conversation_id；由若干轮消息构成
事件（Event）会话内一次可被记录的最小过程单元，拥有唯一 event_id（如 sevt_…），带类型、开始时间、耗时、Token 等元数据
事件类型（Event Type）见第 4 节：用户输入 / 模型思考 / 调用工具 / Skills / MCP / 知识检索 / 模型输出 / 系统信息 / 报错信息
渠道类型（Channel）会话的产生来源，本期枚举：workflow智能体机器人
消耗 Token事件级：该事件的 输入 Token / 输出 Token（如 5.6k / 69）；会话级：全部事件累计的总 Token
耗时（Duration）单个事件从开始到结束的时长（毫秒 / 秒）；会话级为总时长
开始时间（Offset）事件相对会话起点的偏移时间（如 0:00:11），用于还原时间线顺序
会话空闲（Idle）相邻事件之间用户无操作的间隔，事件流中以分隔条标注（如"会话空闲 · 31.6 秒"）
3现状问题与升级总览

❌ 现状（图 4 / 图 5）

  • 列表仅有：会话 ID / Bot / 用户 / 消息数 / 状态 / 创建时间 / 最后答复时间 / 响应时长
  • 详情仅有 USER / ASSISTANT 两条消息 + 右侧基础会话信息
  • 中间过程（思考 / 工具 / 检索 / 报错）完全不可见
  • 无 Token 统计、无逐步耗时、无渠道区分

✅ 升级后（图 1 / 图 2 / 图 3）

  • 列表新增渠道类型、消耗 Token 两列
  • 详情改为结构化事件流：9 类事件按时间线排列
  • 每个事件行标注 Token / 耗时 / 开始时间
  • 支持 events 筛选（默认 All events）+ 搜索
  • 点击事件 → 右侧抽屉展示完整详情（渲染 / 原始 / Deltas）
范围说明
本期聚焦会话列表字段与会话详情事件流两块的采集与展示改造，不改变会话本身的运行机制；埋点采集的实现方案见第 8 节要求。
4事件模型（9 类事件）

会话详情把整段对话拆解为一串结构化事件，每个事件都带统一的元数据（event_id、类型、开始时间、耗时、输入/输出 Token），并按需携带各自的业务字段。事件类型如下：

事件类型角色标签结构化记录内容展示指标
用户输入 用户用户 query 文本、附件（文件/图片）、可选 tenantId 等入参开始时间
模型思考思考模型的思考 / 推理过程文本（thinking / reasoning）Token（入/出）、耗时、开始时间
调用工具工具工具名、输入参数（tool_use）+ 返回结果（tool_result，含来源链接）Token、耗时、开始时间
SkillsSkills技能名、命中的技能指令、输入 / 输出Token、耗时、开始时间
MCPMCPMCP Server、method、请求参数、返回结果Token、耗时、开始时间
知识检索知识检索 query、命中片段、相关性分数、知识库 / 来源Token、耗时、开始时间
模型输出代理人Agent 最终回复（支持 Markdown 渲染，如表格 / 列表 / 链接）Token、耗时、开始时间
系统信息系统系统级事件：会话空闲、上下文压缩 / flush、状态变更、限流等耗时、开始时间
报错信息报错错误码、错误消息、发生阶段、堆栈 / 原始返回，可标记重试耗时、开始时间
采集完整性
"调用工具"需成对记录 tool_use（输入）与 tool_result（输出）并归属同一事件组；报错信息必须独立成事件、不得被吞没在正常输出里，以便排查。

字段状态说明

在现有筛选（Bot / 会话 ID / 用户 / 开始时间 / 结束时间）基础上，新增 渠道类型 筛选（全部 / workflow / 智能体机器人）。其余（刷新 / 重置 / 导出 Excel）保持不变。

6会话详情页改造（事件流）

详情页由"两条消息"改造为结构化事件流时间线，形态参考图 1/2/3。整体分为：①顶部会话概览 → ②筛选/搜索工具条 → ③事件流列表 → ④右侧详情抽屉。

6.1 顶部会话概览（会话元信息区）

顶部这一行不是"并列的核心元素"，而是会话元信息区，由一个身份主键 + 若干归属 / 配置标签构成，服务于「定位 + 归因」两件事——排查问题时，光有会话 ID 只能定位"哪一条"，要判断"为什么这样"必须知道它绑定的 Bot / 渠道 / 知识库。字段拆解：

元素维度回答的问题性质
会话 ID身份标识这是哪一条会话全局唯一主键
Bot / Agent（深度研究员）归属对象这条会话由谁来应答多对一：一个 Bot 下含多条会话
渠道类型（workflow）来源渠道这条会话从哪个入口产生分类标签（见第 5 节）
知识库（事件指挥官资料库）运行配置这条会话用了哪个知识库运行时绑定的配置
层级关系
租户 → Bot（深度研究员）→ 会话（会话 ID）；渠道 / 知识库是该会话运行时绑定的配置。本期「环境」不在详情页透出，避免信息冗余。

此外展示状态、总耗时、总 Token（输入/输出）、创建相对时间。示意：

6.3 Events 筛选（默认 All events）

工具条提供 events 筛选下拉，默认「All events / 所有活动」；可按事件类型多选过滤（如只看「调用工具 + 报错」用于排障，或只看「用户 + 模型输出」还原对话）。可叠加：发言者筛选（用户 / 某 Agent）、关键词搜索。筛选只影响展示，不改变原始记录。

6.4 右侧详情抽屉

点击任一事件行，右侧滑出详情抽屉（参考图 2 / 图 3），展示该事件的完整结构化内容：

  • 头部：事件类型 + 事件名（如 网络搜索 / 信息）、event_id、开始时间、耗时
  • 视图切换：渲染（友好排版，如 Markdown 表格）/ 原始（raw payload）/ Deltas（流式增量，模型输出类事件适用）
  • 调用工具类：分区展示 agent.tool_use（输入查询）与 agent.tool_result（返回结果，含来源链接、可跳转），每段可复制
  • 模型输出类：展示渲染后的完整回答；报错类：展示错误码、消息、堆栈；知识检索类：展示命中片段与来源

7指标口径与展示规范
指标口径定义展示格式
Token（事件级）该事件的输入 Token / 输出 Token5.6k / 69（入 / 出）
Token（会话级）会话内所有事件 Token 累计列表列展示总量；详情头 167k / 4.7k
耗时事件从开始到结束时长；<1s 用毫秒，≥1s 用秒827ms / 7.5s / 2分59秒
开始时间相对会话起点的偏移0:00:11（时:分:秒）
会话空闲相邻事件间隔（用户无操作）分隔条「会话空闲 · 31.6 秒」
缺失兜底
某类事件不产生 Token（如纯用户输入 / 系统信息）时，Token 位留空不显示，不显示 0，避免误读。
8数据采集要求（研发对齐）
  • 全过程落库：运行时把 9 类事件按发生顺序写入会话事件表，字段含 event_id / conversation_id / type / role / start_offset / duration / token_in / token_out / payload。
  • 渠道标记：会话创建时写入 channel（workflow / 智能体机器人），供列表列与筛选使用。
  • 成对记录：工具 / MCP / Skills 的调用与返回成对入库并可关联为同一事件组。
  • 报错不吞没：任何阶段异常都单独记为 error 事件，保留错误码与原始返回。
  • 原始可回放：payload 同时保留 渲染所需结构 与 raw 原文，支撑抽屉「渲染 / 原始 / Deltas」切换。
  • Token 汇总：会话级总 Token 由事件级累加得出，保证列表与详情口径一致。
9边界与非目标

✅ 本期做

  • 列表新增 渠道类型 / 消耗 Token 两列及筛选
  • 详情事件流 + 逐事件 Token/耗时/开始时间
  • events 筛选（默认 All events）+ 搜索
  • 右侧详情抽屉 + 渲染/原始/Deltas

🚫 本期不做

  • 不改会话的运行 / 调度机制本身
  • 不做跨会话的聚合报表 / 成本看板（后续）
  • 不做实时告警 / 异常自动通知（后续）
  • 渠道枚举本期仅 workflow / 智能体机器人
10验收标准
  1. 列表页可见并可筛选渠道类型，可见消耗 Token 且与详情总量一致。
  2. 详情页能完整回放一次含工具调用 / 知识检索 / 报错的会话，9 类事件均正确归类。
  3. 每个事件行正确显示 Token / 耗时 / 开始时间；空闲分隔条正确。
  4. events 筛选默认 All events，切换后事件流按类型正确过滤。
  5. 点击事件右侧抽屉展示完整详情，且「渲染 / 原始」可切换、内容可复制。
  6. 报错事件独立可见，含错误码与原始返回，可用于排查。
