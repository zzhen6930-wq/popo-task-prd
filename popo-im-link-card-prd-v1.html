POPO-CLI · IM Skill 链接卡片化

让 popo-cli IM 发出的链接，像 POPO 客户端一样自动解析成卡片
📅 2026-06-22 🏷️ v1.0 👤 作者：张振 🧩 归属：POPO-CLI / popo-im Skill
1背景与问题

POPO 客户端发链接时会自动把链接解析成卡片，对内部和外部链接有两种不同形态：

  • 内部云文档链接 → 展示为文档卡片，支持设置「可阅读 / 可编辑」权限
  • 外部链接（如公众号文章、外网页面）→ 展示为预览卡片（标题、摘要、来源），不涉及权限

但目前通过 popo-cli 的 IM Skill 发出的链接不会解析，对方收到的是一段裸 URL 文本，导致：

  • 内部云文档无法在发送时直接设权限，发出后还要再手动调整
  • 外部链接没有标题预览，对方点开前不知道是什么内容
  • 同样一条消息，从客户端发和从 CLI 发，呈现效果不一致
2用户期望

❌ 现状（popo-cli 发链接）

消息内容里只有一段纯文本 URL，对方需要点开才能知道是什么；云文档也没有权限提示。

✅ 目标（与 POPO 客户端一致）

链接自动解析为卡片：内部云文档显示文档名+可设权限；外部链接显示标题+摘要+来源。

💡 一句话需求
popo-cli IM Skill 发出的链接也支持解析，效果与 POPO 客户端完全一致。
3两种链接类型的处理
📄 内部云文档链接
域名指向 POPO 云文档（如 docs.popo...）
解析为：文档卡片（文档图标 + 文档名）
支持：权限设置 → 可阅读 / 可编辑
默认权限：可阅读（与客户端一致）
🌐 外部链接
非 POPO 云文档的所有 URL（公众号、外网等）
解析为：预览卡片（标题 + 摘要 + 来源域名 + 缩略图）
不涉及权限设置
解析失败时：保留为纯文本 URL（不阻塞发送）
4用户场景

场景 1：发内部云文档给群

  • 用户："把这份 PRD（云文档链接）发到 X 群，可编辑权限。"
  • 对方在群里看到：文档卡片 + 群权限：可编辑（与客户端发出来的完全一样）。

场景 2：发外部链接给同事

  • 用户："把这篇公众号链接发给李四。"
  • 李四收到：带标题和摘要的卡片，能直接看到是什么内容。

场景 3：消息里链接 + 文本混排

  • 用户："在 X 群发：『大家明天记得在表格里补充：[云文档链接]』"
  • 对方看到：文字 + 文档卡片，效果与客户端发出来的一致。
5一期范围
能力是否在一期
内部云文档链接自动解析成文档卡片✅ 做
云文档卡片支持设置「可阅读 / 可编辑」权限✅ 做
外部链接自动解析成预览卡片（标题/摘要/来源）✅ 做
消息里链接 + 文本混排✅ 做
解析失败时降级为纯文本 URL（不阻塞发送）✅ 做
除「可阅读 / 可编辑」外的其他权限粒度（如指定人）❌ 不做
主动撤回 / 修改已发卡片的权限❌ 不做
6交付要求
  • 用户在 Agent 中说「把这个链接（具体 URL）发给 XX」「在 X 群发这条云文档链接」时，链接默认就被解析，无需用户额外指定开关
  • 云文档默认权限=可阅读，用户明确说「可编辑」时切换
  • 对方端最终看到的效果 = 客户端发出的卡片效果，不能有视觉/能力差异
