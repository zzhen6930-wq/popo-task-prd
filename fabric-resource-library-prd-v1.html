WorkHub Studio · AI 工厂「资源库」

把 Agent 的结果交付物沉淀成可查找、可复用、多端可取的「资源库」，让产出不再随对话消失。
📅 2026-07-23 🏷️ v1.0 👤 作者：青远（张振） 🧩 归属：WorkHub Studio / AI 工厂 · 资源库
1背景与目标

当前 WorkHub Studio 里，Agent 生成的产出物没有被持久保存——报告、表格、图片、代码、生成的网页等都散落在一次次对话里，导致：

  • 找不回：产出物随会话沉底，事后想再用只能翻聊天记录，甚至已丢失
  • 难复用：无法集中管理、检索、二次编辑或分享
  • 多端断裂：换个端（Web / 客户端 / 移动）就拿不到之前的产出
  • 无沉淀：有价值的成品无法收藏、归档，形成不了个人 / 组织资产

本期目标：新增「资源库」能力——把 Agent 产出的结果交付物自动入库，统一按文件名 / 来源 / 时间管理，支持收藏、搜索、筛选、预览、下载、来源跳转，并多端可取。

🔑 一句话定位
给 Agent 的产出装一个「成品仓库」：只收最终交付物，过程草稿不进；一次生成，处处可取、随手可藏。
2核心概念与术语
术语说明
资源库（Library）集中存放 Agent 结果交付物的空间，以文件 / 云空间链接为条目，支持管理与检索
结果交付物Agent 面向用户最终交付的成品（可带走、可使用），入库对象
过程产物任务中间态：草稿、临时缓存、调试信息、工具原始返回、被覆盖的旧版本等，不入库
来源（Source）该交付物由哪个会话 / Agent / 自动化任务产生，记录并可跳转回溯
收藏（Favorite）用户对条目打星标，聚合到「收藏」视图，便于常用产物快速取用
云空间产物以链接形式入库的 POPO 云文档 / 多维表 / 云文件，只存链接与元信息，点击跳转
3入库规则（只收结果交付物）

入库的核心边界：只有结果交付物才入库，过程中的产物不入库。判定原则如下：

判定维度结果交付物（✅ 入库）过程产物（🚫 不入库）
是否最终输出任务的最终成品，作为答复交付给用户中间步骤生成的临时件
是否可带走使用用户可下载 / 打开 / 分享 / 二次编辑仅供内部推理、调试查看
典型举例最终报告 PDF、生成的 PPT、交付代码 zip、成图 / 视频、生成的 HTML 应用、POPO 文档链接思考草稿、工具原始返回、调试日志、被覆盖的旧版本、临时截图
入库时机与元信息
Agent 产出结果交付物时自动入库，记录元信息：文件名、类型、来源（会话/Agent/任务 + 可跳转）、生成时间、大小、存储地址。同一交付物多次迭代时以最新版本为准（历史版本策略后续再议）。
判定兜底
当无法明确区分时，遵循「宁缺勿滥」——只把明确作为交付答复的产物入库，避免过程噪声污染资源库。
4支持入库的文件类型

按功能用途分为 6 大本地文件类别 + 1 类云空间产物：

📄 1. 文档与文字处理类
Markdown .md、PDF .pdf、Word .docx、纯文本 .txt、Typst .typ
报告、记录、正式交付
🌐 2. 网页与交互应用类
HTML .html、CSS .css、JavaScript .js、TypeScript .ts
动态展示、小型 Web 工具
📊 3. 数据与电子表格类
Excel .xlsx .xls、CSV .csv、JSON .json、XML .xml、SQLite .sqlite .db
存储、分析、结构化数据交换
💻 4. 代码与开发脚本类
Python .py、Shell .sh、Java / C++ / Go / Rust 等源码、Dockerfile / YAML .yaml .yml、压缩包 .zip .tar.gz
自动化工具、分析脚本、完整项目
🎨 5. 视觉、多媒体与设计类
图片 .png .jpg .jpeg .webp、矢量图 .svg、音频 .mp3 .wav、视频 .mp4、演示文稿 .pptx
AI 生成图、图表、TTS、视频、幻灯片
📐 6. 图表与架构设计类
Mermaid .mmd、D2 / PlantUML .d2 .puml
流程图、时序图、甘特图、架构图源码
☁️ 7. 云空间产物（以链接入库）
POPO 云文档链接、POPO 多维表 / 电子表格链接、POPO 云文件（PDF / Excel / Word / zip 等）链接
只存链接与元信息，不落本体，点击跳转 POPO 打开
类型识别
入库时按扩展名 / MIME 识别类型并归类，列表用类型图标区分；未知类型按「其他文件」兜底保存，不阻断入库。

5.1 字段说明

字段说明
文件名带类型图标 + 文件名；已收藏显示 ★ 标记；html / POPO 云文档等直开类型显示「↗ 浏览器打开」标记
来源产生该产物的会话 / Agent / 自动化任务；点击跳转到「对话页」并自动锚定该产出物位置（见 6 节）
创建人产生该产物的用户，展示用户邮箱
大小文件大小；云空间产物显示「链接」
时间入库 / 生成时间，支持按时间排序
操作行内直接可操作三项：📍 在任务中定位、⬇ 下载、★ 收藏 / 取消收藏

5.2 视图 / 搜索 / 筛选

  • 全部 / 收藏 Tab：全部=所有入库条目；收藏=打星标的条目聚合
  • 搜索：按文件名 / 来源 / 创建人关键词检索
  • 筛选：按文件类型（7 类）、来源、时间范围过滤，可组合
6交互能力
能力说明
收藏行内点击 ★ 即时收藏 / 取消，收藏态多端同步
预览（点击行）点击列表行即预览，采用大预览（居中大弹窗，白纸张式内容区）：图片放大看、md / 代码渲染、docx / pdf 内嵌预览、音视频播放。例外：html、POPO 云文档等不进后台预览，点击直接在浏览器打开原文件
在任务中定位 📍点击来源或行内📍 定位，均跳转到「对话页」并自动锚定该产出物在对话中的位置，回到产生上下文
下载 ⬇本地文件类可直接下载原文件；云空间产物跳转到 POPO 下载
多端可取库数据云端统一存储，Web 后台 / 客户端 / 移动端一致访问
7边界与非目标

✅ 本期做

  • 结果交付物自动入库（6 类本地 + 云空间链接）
  • 资源库列表：全部 / 收藏、搜索、筛选
  • 收藏、预览、下载、来源跳转、重命名 / 删除
  • 多端可取

🚫 本期不做

  • 不做过程产物入库
  • 不做版本历史管理（仅留最新版本，后续）
  • 不做多人协作 / 权限共享（后续）
  • 不做库内二次编辑（预览为主，编辑跳原生工具）
8验收标准
  1. Agent 产出结果交付物时自动入库，过程产物不入库。
  2. 支持第 4 节列出的全部文件类型正确识别与归类；未知类型兜底不阻断。
  3. 列表按 文件名 / 来源 / 创建人（邮箱）/ 时间 展示，全部 / 收藏 Tab 可切换，收藏可即时 toggle。
  4. 操作三项行内直接可用：📍 在任务中定位 · ⬇ 下载 · ★ 收藏/取消收藏。
  5. 搜索 / 筛选（类型 / 来源 / 时间）生效。
  6. 点击行大预览；html / POPO 云文档点击直接在浏览器打开，不进后台预览。
  7. 点击来源 / 定位能跳转「对话页」并锚定该产出物位置。
  8. 云空间产物以链接入库，点击跳转 POPO 打开。
  9. 库数据多端一致可取。
