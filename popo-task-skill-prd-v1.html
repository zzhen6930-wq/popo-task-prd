POPO-CLI · popo-task Skill（一期 MVP）

把 POPO 客户端「任务」模块的核心能力，搬到本地 Agent 对话里
📅 创建：2026-06-22 🏷️ 版本：v1.0 👤 作者：张振 🧩 归属：Ⓑ POPO-CLI Skills
1背景与目标

POPO 客户端「任务」模块是员工日常协同的高频入口，承载任务创建、跟进、查看等核心场景。POPO-CLI 当前已上线 IM、云文档、电子表格、多维表格、日程/会议室共 5 个 Skills，任务能力缺失，导致用户无法在 Codemaker、龙虾等本地 Agent 中完成任务相关诉求，必须切回 POPO 客户端。

本期目标：上线 popo-task Skill，让用户在与 Agent 的自然对话中即可完成创建任务与查看任务两类最高频操作，对齐 POPO 客户端「任务」模块的核心能力布局。

📌 设计基准
本 PRD 所有功能项与 POPO 客户端「任务」页面一一对齐，不引入客户端尚未提供的概念。
2一期范围（做 vs 不做）

✅ 一期要做

  • 创建任务（可填写任务字段、可指定所属项目）
  • 更新任务（修改任务字段：执行人、时间、项目、优先级、备注等）
  • 删除任务
  • 创建子任务（在已有任务下挂子任务）
  • 查看任务详情（含执行人完成进度、附件、备注、子任务等）
  • 查看评论与查看日志（任务变更操作记录）
  • 日志中由 Agent 触发的创建 / 更新条目，文字后追加「✦ AI 标识」（对齐日程 skill 的现有交互）
  • 上传附件到任务（对齐客户端「附件 · 添加附件」入口）
  • 查看「全部任务」
  • 查看 3 个固定「快捷方式」视图：分配给我、我关注的、我创建的
  • 查看具体某个已有「项目」下的任务
  • 应用 4 个「快捷筛选条件」：已完成、分配给我、今天截止、本周截止
  • 应用 8 个「添加条件」自定义筛选维度
  • 查询结果支持分组展示

❌ 一期不做

  • 不支持新建项目（只能在已有项目下建任务）
  • 不做看板、日历、时间线视图的 CLI 等价物（这些是客户端可视化形态，CLI 一期只输出列表）
  • 不做评论的写操作：添加评论、回复评论、删除评论（仅支持查看评论；写操作涉及富文本、@ 提及、权限校验，复杂度高，整体放二期）
  • 不做子任务的智能拆解（客户端「智能拆解」按钮能力，一期不在 CLI 提供）
  • 不做附件的下载（客户端任务详情面板未提供下载入口，CLI 保持一致；如需下载请在客户端打开）
  • 不做任务变更的事件订阅与推送
💡 一期原则
先把 POPO 客户端任务模块里使用频次最高的「建 + 看」这两件事在 CLI 里跑通，覆盖率优先于功能纵深。
3核心能力
📝
创建任务
自然语言描述任务，Agent 调用本 Skill 完成创建，可指定字段，可挂到已有项目下，可挂子任务。
✏️
更新 / 删除任务
修改任务字段（执行人、时间、项目、优先级、备注等）；删除任务。
📋
查看任务
「全部任务 / 快捷方式 / 项目」三种入口；叠加快捷筛选、自定义筛选、分组；查看单条任务的详情、评论、日志。
🔍
详情 / 评论 / 日志
查看单条任务的完整详情、所有评论（一期只读）、所有变更日志，回答「这个任务进展到哪儿了」类问题。
🛡️ 全局写操作约束：执行前必须用户确认
创建任务、创建子任务、更新任务、删除任务均为写操作。Skill 在执行前必须：
  1. 向用户回显将要执行的内容（操作类型 + 目标任务 + 涉及字段及取值）
  2. 等待用户在对话中明确确认（如「确认」「执行」「OK」等）
  3. 用户确认后方可真正调用接口；若用户取消，整条操作中止
该约束对所有写操作生效，本 PRD 后续章节不再单独重复说明。
📎 设计参考：popo-calendar 仅对创建/取消/退出/接受拒绝邀请做二次确认，更新日程直接执行。popo-task 选择更保守的策略——更新任务同样要求确认，原因：任务字段修改（如执行人、截止时间）会通知到执行人和关注人，影响面与新建相当，宁可多一步确认以防误改。
4创建任务

4.1 支持的任务字段

字段口径与 POPO 客户端「新建任务」一致。一期 只有「任务名称」必填，其余字段按用户表达自动带入：

字段必填说明
任务名称是对齐客户端「输入任务名称，回车保存」
执行人否支持多人；任务详情页展示「X/Y 人完成」进度；缺省=创建人本人
关注人否支持多人
截止时间否支持自然语言（如「明天」「下周一」「6 月 30 日」）
开始时间否同上
提醒时间否同上
优先级否沿用客户端优先级选项
所属项目否缺省=不归属任何项目（落在「无」分组下）
备注否对齐客户端详情面板「备注 · 编辑」
附件否对齐客户端「附件 · 添加附件」：一期支持上传附件、展示已有附件列表；不支持下载（与客户端一致，客户端目前也不提供下载入口）

4.2 项目选择规则

  • 用户在对话里指定项目名 → 在用户可见的已有项目中匹配
  • 匹配唯一 → 直接归属到该项目
  • 匹配多个 → Agent 列出候选让用户确认
  • 未匹配到 → 明确提示「未找到该项目」，引导用户到 POPO 客户端先建项目
⚠️ 项目能力边界
一期 不在 CLI 中提供「新建项目」入口。原因：项目的创建涉及成员、关联群组等管理动作，更适合在客户端图形化完成。

4.3 用户场景示例

「帮我建个任务：完成 popo-task PRD 评审，明天下班前交，分配给李四，挂到 POPO AI 能力推进项目下。」
→ Agent 先回显：「将创建任务【完成 popo-task PRD 评审】 / 项目=POPO AI 能力推进 / 执行人=李四 / 截止=明天 18:00，是否确认？」
用户确认后才真正调用 popo-task 创建任务，回执任务链接。
「记一条待办：周五前看完云协作 UX 计划表。」
→ 回显待创建任务信息（截止=本周五，无项目，执行人=用户本人），确认后创建。
5查看任务

客户端「任务」模块的查询能力由四个层次叠加而成，本 Skill 完整对齐：

  1. 入口：选择从哪里开始看（全部 / 某个快捷方式 / 某个项目）
  2. 快捷筛选条件：一键快速过滤
  3. 添加条件（自定义筛选）：组合更精细的条件
  4. 分组：把结果按某个维度归类

5.1 入口（必选其一）

📂 全部任务
客户端左上角默认项
  • 当前用户可见的所有任务
⚡ 快捷方式（3 项）
客户端左侧固定视图
  • 分配给我
  • 我关注的
  • 我创建的
📁 项目视图
客户端左侧「项目」区
  • 查看用户的某个具体项目下任务（如「青远的测试项目」）

5.2 快捷筛选条件（4 项，可叠加在任意入口上）

对齐客户端筛选弹窗顶部的「快捷筛选条件」区：

已完成 分配给我 今天截止 本周截止
💡 与"显示已完成"开关的关系
客户端列表上方有「显示已完成」开关。一期处理为：默认不展示已完成任务；用户明确表达「看已完成」时（如使用「已完成」快捷筛选或自然语言"我已经完成的"），Skill 自动包含已完成任务。

5.3 添加条件（8 项自定义筛选，可叠加）

对齐客户端筛选弹窗「添加条件」下拉列表，完整 8 项：

执行人 创建人 关注人 截止时间 开始时间 提醒时间 所属项目 优先级

多个条件之间为 AND 关系（与客户端一致）。

5.4 分组展示

支持按以下维度对查询结果分组（对齐客户端列表分组能力，例如客户端默认按「无 / 项目」分组）：

  • 所属项目
  • 执行人
  • 优先级
  • 截止时间（今日 / 本周 / 本月 / 更晚 / 无）

5.5 用户场景示例

「我今天有哪些任务？」
→ 入口=分配给我 + 快捷筛选=今天截止。
「青远的测试项目下还有哪些紧急任务没做完？」
→ 入口=项目「青远的测试项目」 + 添加条件=优先级=紧急 + 默认不含已完成。
「我创建的任务里，本周要截止的有哪些，按项目分组。」
→ 入口=我创建的 + 快捷筛选=本周截止 + 分组=所属项目。
「李四这周开始要做的所有任务。」
→ 入口=全部任务 + 添加条件=执行人=李四 + 添加条件=开始时间=本周。
6任务详情：详情、评论、日志

对齐 POPO 客户端任务详情侧边栏。用户对单条任务表达「看下详情 / 看下进度 / 看下大家怎么说 / 看下谁改过什么」时，Skill 提供对应能力。

6.1 查看任务详情

返回客户端详情面板上能看到的所有信息：

  • 任务名称、所属项目、执行人（含「X/Y 人完成」进度）
  • 时间（开始 / 截止 / 提醒）、优先级
  • 备注、附件清单（展示文件名；如需查看附件内容请在客户端打开）
  • 子任务列表（含每个子任务的名称、执行人、截止时间、完成状态）
  • 评论数、日志数（提示用户可继续展开）

6.2 查看评论

对齐详情面板「评论」Tab。一期仅支持「读」：展示该任务下所有评论，每条评论包含评论人、发布时间、评论内容、是否为回复（以及回复的对象）。

⚠️ 一期不做评论写操作
添加评论、回复评论、删除评论均不在一期范围。评论写操作涉及富文本、@ 提及、权限校验等，复杂度高，整体放二期。

6.3 查看日志

对齐详情面板「日志」Tab。展示该任务的变更记录，例如：

  • 「利奥李(李扬) 06/17 19:02 · 完成了任务」
  • 「qingtian(李斌斌) 05/13 13:47 · 关注了该任务」
  • 「魏建磊 05/13 13:46 · 关注了该任务」

用户可用自然语言问「这个任务都谁改过？」「最近谁完成了？」，Agent 基于日志回答。

6.4 AI 标识（重点变更）

参考 POPO 客户端日程模块的「日程修改 · AI 标识」交互：用户通过 Agent（popo-task Skill）完成的创建任务、更新任务，在「日志」中对应那条记录的文字后面追加一个 ✦ AI 标识图标，hover 时浮层提示「本次修改由 AI 操作」。

  • 位置：紧跟在动作文字后面（例「创建了任务 ✦」「更新标题为 XXX ✦」），不影响时间戳和用户名样式
  • 覆盖范围：通过 popo-task Skill 触发的创建任务、创建子任务、更新任务所产生的日志条目
  • 客户端原生操作产生的日志条目不带此标识
  • 删除任务：任务被删除后已无详情入口，无需考虑日志展示（如未来支持「已删除」回收站再处理）
💡 价值
让任务的修改来源对所有协作者透明：用户能一眼分辨哪些变更是人工操作、哪些是通过 AI 完成，便于协作中追溯与责任界定。

6.5 用户场景示例

「『请在本周完成个人目标设置提交』这个任务，现在多少人完成了？」
→ 查看任务详情，返回执行人完成进度（如 8/37 人完成）。
「测试项目下的测试任务的评论里大家说了啥？」
→ 查看评论 Tab，按时间倒序输出。
7更新任务 / 删除任务 / 创建子任务

7.1 更新任务

用户在对话中指明要改的任务和要改的字段，Skill 完成更新。可更新字段范围 = 第 4 章「支持的任务字段」一致：

  • 任务名称、执行人、关注人、截止/开始/提醒时间、优先级、所属项目、备注
  • 支持单字段更新，也支持多字段同时更新
「把『测试项目下的测试任务』的截止时间改成下周一，再把李四加进执行人。」
→ Agent 回显：「将更新任务【测试项目下的测试任务】 / 截止时间：6 月 16 日 → 下周一 / 执行人：青远(张振) → 青远(张振)、李四，是否确认？」
用户确认后执行更新。

7.2 删除任务

对齐客户端详情页右上角的删除入口。除「全局写操作约束」要求的二次确认外，删除场景还需额外做到：

  • 回显内容包含任务名称、所属项目、执行人，方便用户核对
  • 若该任务存在子任务，需提示用户「将一并删除其下 N 个子任务」
  • 删除为不可逆操作，文案需明确

7.3 创建子任务

对齐客户端「在任务下新建子任务」能力（鼠标悬停任务行右侧出现「新建子任务」入口，或在详情面板「子任务 · 新建子任务」处操作）。

  • 必须指定挂载到哪个父任务下
  • 子任务字段范围与普通任务一致（一期允许最少只填名称）
  • 父任务的所属项目自动继承到子任务
「在『测试项目下的测试任务』下拆 3 个子任务：写文档、约评审、跟进结论，都分给我。」
→ Agent 回显将创建的 3 条子任务清单（含父任务、执行人=当前用户、继承的项目），用户确认后批量创建。
⚠️ 一期不做子任务智能拆解
客户端详情面板「子任务 · 智能拆解」按钮的能力一期不在 CLI 提供；若用户表达「帮我智能拆一下」，引导其到客户端使用。

9二期方向（不在一期范围）

  • 附件的下载（待客户端先放开下载入口后再跟进）
  • 任务变更事件订阅，让 Agent 主动提醒用户
  • 面向高频指令的 Shortcut（如「我今天的待办」「我本周的任务摘要」）
