Fabric · AI 工厂「定时任务」菜单（一期 MVP）

让组织配置者能给 Agent 配置周期性自动任务，覆盖运维日报、定时播报等场景
📅 2026-06-23 🏷️ v1.0 👤 作者：张振 🧩 归属：Fabric / AI 工厂
1背景与目标

Fabric 是面向组织/租户的 Agent 配置平台。当前 Agent 只能在用户对话时被动触发执行，缺少主动 / 周期性触发能力，典型场景跑不通：

  • 每天 18:00 让 Agent 把"群运维日报"主动推给负责人 → 跑不通
  • 每周一早上 9:00 让 Agent 在指定群里发"本周提醒" → 跑不通
  • 财税合规 Agent 每月 28 日提醒一次"个税申报截止" → 跑不通

本期目标：在「AI 工厂」下新增「定时任务」菜单，让组织配置者能给 Agent 配置周期性自动任务。结合已有内置工具「机器人主动发消息」，完整支撑"定时触发 → Agent 执行 → 机器人送达"的端到端链路。

2产品定位

2.1 谁来配、谁受益

角色能做什么
组织配置者（管理员）在「AI 工厂 · 定时任务」菜单下配置、启停、查看历史
使用者（普通成员）本期不可配置定时任务；只作为被触发动作的接收方（消息接收、报告查看）

2.2 为什么放在「AI 工厂」菜单，而不是做成内置工具

维度菜单方案（本期）工具方案（一期不做）
配置入口后台表单对话（Agent 调用工具）
谁能配组织配置者使用者
本期核心场景匹配度✅ 运维日报等组织级场景❌ 个人级提醒为主
使用者侧承接面不需要需要"我的提醒"等承接（Fabric 当前没有）
风险面受管理员管控，可控使用者可配，需做配额/审计
💡 与「内置工具」模式互补
内置工具模式（类似 openclaw "对话即设定提醒"）是个优雅设计，但前提是平台已有使用者侧的提醒承接面。Fabric 当前没有，放二期再补。一期把组织级跑通，底层调度引擎统一，未来加工具入口零迁移成本。

4一期范围

✅ 一期要做

  • 新增「定时任务」一级菜单（仅组织配置者可见）
  • 定时任务列表页：查看 / 创建 / 编辑 / 启停 / 删除
  • 定时任务详情页：基础信息 + 执行历史
  • 触发频率支持：每天 / 每周 / 每月 + 具体时间点
  • 执行内容：选择一个 Agent + 给 Agent 的指令（Prompt 模板）
  • 启停状态切换
  • 执行失败后自动重试 1 次；仍失败则记录并告警给配置者

❌ 一期不做

  • 完整 cron 表达式（一期只用"每天/每周/每月+时间点"的预设）
  • 使用者侧配置入口（即对话式设置定时，留二期 + 工具模式）
  • 任务之间的依赖、串联
  • 动态参数（如"上周一到本周日"等运行时计算的时间段，一期由 Prompt 模板内描述）
  • 外部 webhook 触发（非时间触发）
  • 配额管理后台（按需在二期补）
5定时任务配置项
字段必填说明
任务名称是便于配置者识别，1~40 字
描述否说明该任务用途
关联 Agent是从本组织已有 Agent 中选择一个；Agent 启用 / 删除后联动校验
触发频率是枚举：每天 / 每周（指定 1~多个工作日）/ 每月（指定日期，含"最后一天"）
触发时间是具体到分钟（如 18:00）；时区默认按组织时区
给 Agent 的指令是Prompt 模板，例如「请汇总今天 POPO 用户反馈群里的关键问题，整理成日报后通过机器人主动发给 @张振」
启用状态是启用 / 停用，默认启用
💡 与「机器人主动发消息」工具协作
定时任务只负责触发 + 喂指令，"消息送达"由 Agent 调用机器人主动发消息工具完成。配置者需保证：① Agent 已挂载该工具；② Agent 已绑定有效机器人；③ 指令里说清"发给谁"。

  • 下半部分：执行历史列表，按时间倒序，每行展示：执行时间、耗时、结果（成功/失败）、错误原因（失败时）、查看执行日志
  • 详情页可直接编辑配置（修改后立即生效，下次按新配置触发）

失败重试与告警

  • 单次执行失败 → 自动重试 1 次（间隔 1 分钟）
  • 重试仍失败 → 记录到执行历史，标红显示，并通过 Fabric 站内消息告警给该任务的配置者
  • 连续 3 次执行失败 → 自动停用该任务，避免持续无效消耗
8典型场景串联
场景 1 · 群运维日报
每晚 18:00 把当天运维情况发给负责人
① 在「定时任务」配置：关联「群运维 Agent」/ 每天 18:00 / 指令="请汇总今日 POPO 用户反馈群里的关键问题，整理成日报后用机器人发给 @张振"
② 18:00 调度器触发 → Agent 收到指令 → 调用机器人主动发消息工具 → 张振 IM 收到日报。
场景 2 · 月度合规提醒
财税合规 Agent 每月 28 日提醒
财务部门管理员配置：关联「财务审核助手」/ 每月 28 日 09:00 / 指令="在财务公告群里发送：本月个税申报将于 5 个工作日内截止，请同学们准备好材料。"
场景 3 · 周一团队早会提醒
每周一早上 9:00 推送本周日程概要
关联「团队周报 Agent」/ 每周一 09:00 / 指令="拉取本周所有关键日程，整理成简报并发到团队群。"
9权限与依赖
  • 菜单可见性：仅组织配置者（管理员）可见 / 可操作；普通使用者完全不可见
  • 对 Agent 的依赖：任务关联的 Agent 被删除 → 任务自动停用并提示
  • 对工具的依赖：若指令期望调用"机器人主动发消息"工具，但 Agent 未挂载该工具或未绑定机器人 → 执行失败时给出明确原因（依赖第 7 节告警通路）
  • 时区：默认采用组织时区；一期不开放任务级时区设置
10交付与验收
  • 菜单出现在 AI 工厂下，仅管理员可见
  • 「POPO 用户群运维日报」场景可以端到端跑通：每天 18:00 自动给负责人发出日报
  • 任务停用 / 启用 / 编辑 / 删除均可正常生效
  • 执行成功 / 失败均有完整历史记录可查
  • 失败重试、告警、连续失败自动停用机制按 7 节描述工作
  • Agent 被删除后，关联的定时任务自动停用，并在列表中给出标记
11二期方向（不在一期范围）
  • 内置工具「设置提醒」（openclaw 模式）：使用者通过对话即可配置个人级定时
  • 使用者侧承接面：「我的提醒」/「我的订阅」列表（前置依赖）
  • 完整 cron 表达式支持
  • 外部事件触发（webhook / 业务事件）
  • 任务运行时动态参数（如"上周一 ~ 本周日"）
  • 配额管理与计费归属（按 Agent / 按组织）
