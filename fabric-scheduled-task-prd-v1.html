WrokHub Studio · AI 工厂「自动化任务」菜单（一期 MVP）

让 B 端租户与 C 端用户都能给 Agent 配置自动化任务，覆盖定时播报、事件响应、外部 WebHook 触发等场景
📅 2026-06-23 🏷️ v2.0 👤 作者：张振 🧩 归属：WrokHub Studio / AI 工厂
1背景与目标

WrokHub Studio 是面向组织/租户的 Agent 配置平台。当前 Agent 只能在用户对话时被动触发执行，缺少主动 / 自动化触发能力，典型场景跑不通：

  • 每天 18:00 让 Agent 把"群运维日报"主动推给负责人 → 跑不通（定时触发）
  • 指定云文档一有变更，就让 Agent 总结变更并通知相关人 → 跑不通（事件触发）
  • 外部系统构建完成后回调一个地址，触发 Agent 生成发布播报 → 跑不通（WebHook 触发）

本期目标：在「AI 工厂」下新增「自动化任务」菜单，让 B 端租户与 C 端用户都能给 Agent 配置自动化任务，支持定时 / 事件 / WebHook 三种触发类型、手动 / 对话两种创建方式。结合已有内置工具「机器人主动发消息」，完整支撑"触发 → Agent 执行 → 推送送达"的端到端链路。

2产品定位

2.1 支持 B 端与 C 端两种场景

自动化任务同时面向 B 端租户与 C 端用户，二者能力一致、共用同一套调度引擎与列表。不区分创建场景，任务列表只记录创建人字段用于识别归属。

场景说明
B 端租户租户下成员为组织级 Agent 配置自动化任务（如运维日报、月度合规提醒）
C 端用户个人用户为自己使用的 Agent 配置自动化任务（如每日资讯汇总、个人提醒）
💡 不区分场景，只记录创建人
系统不需要区分任务是 B 端还是 C 端场景创建的；列表中统一记录创建人即可，数据可见性按创建人所属租户 / 个人空间隔离。

2.2 支持手动创建与对话创建两种方式

创建方式入口说明
手动创建后台页面在「自动化任务」菜单点击"新建"，通过触发器配置抽屉填写表单创建
对话创建与 Agent 对话用户对 Agent 说"帮我创建一个自动化任务"，Agent 触发后回吐表单卡片，用户在卡片内配置触发方式、执行动作、时间/频率等条件后提交创建

4一期范围

✅ 一期要做

  • 新增「自动化任务」一级菜单，B 端租户与 C 端用户均可用
  • 支持手动创建（后台表单）与对话创建（表单卡片）两种方式
  • 支持三种触发类型：定时触发 / 事件触发 / WebHook 触发
  • 任务列表页：查看 / 创建 / 编辑 / 启停 / 删除，记录创建人
  • 任务详情页：基础信息 + 执行历史
  • 执行内容：关联一个 Agent + 给 Agent 的指令（Prompt 模板）
  • 推送配置：推送渠道（关联 Agent / Agent 绑定的 bot）+ 推送对象（部门 / 人 / 群）
  • 静默条件：满足条件则跳过本次推送
  • 执行生成一条执行结论，记录列表标题为「自动化：任务名称」
  • 执行失败后自动重试 1 次；仍失败则记录并告警给创建人

❌ 一期不做

  • 完整 cron 表达式（定时触发只用预设频率 + 时间点）
  • 任务之间的依赖、串联、编排
  • 事件触发的多级复杂条件组合（一期支持单层下级条件）
  • 运行时动态参数（如"上周一到本周日"等，由 Prompt 模板内描述）
  • 配额管理与计费后台（按需在二期补）
5自动化任务配置项

5.1 基础字段

字段必填说明
任务名称是便于识别，1~40 字
描述否说明该任务用途
创建人系统自动记录，用于列表展示与数据隔离；不区分 B/C 端场景
关联 Agent是从可用 Agent 中选择一个；Agent 启用 / 删除后联动校验
给 Agent 的指令是Prompt 模板，例如「请汇总今天 POPO 用户反馈群里的关键问题，整理成日报」
触发方式是三选一：定时触发 / 事件触发 / WebHook 触发（详见 5.2）
推送渠道是见 5.3
推送对象是见 5.3
静默条件否见 5.4
启用状态是启用 / 停用，默认启用

5.2 触发方式（三种）

① 定时触发

子项说明
触发频率预设枚举：仅一次（指定未来某时间点执行一次）/ 每隔几分钟 / 每隔几小时 / 每个工作日（周一到周五）/ 每天 / 每周（指定星期几）/ 每双周 / 每月（指定日期，含"最后一天"）
触发时间具体到分钟（如 18:00）；"每隔几分钟/小时"类型填间隔值；时区默认按创建人时区

② 事件触发

子项说明
选择事件从事件列表中选择，按类目分组：消息（收到聊天消息）/ 日程（日程变更）/ 会议（会议结束、妙记生成、纪要生成）/ 文档（云文档变更、收到评论、被 @、多维表记录变更）/ 邮箱（收到新邮件）等
下级条件选择事件后，出现下级条件入口：
· 需指定对象的事件（如"云文档变更"）→ 出现「指定云文档」选择器
· 消息类事件 → 出现「筛选条件」（指定群组 / 指定发送人 / 含任一关键词 / 含全部关键词 / 只接收 @我的消息 / 单聊）

③ WebHook 触发

子项说明
WebHook 地址系统生成的 POST 地址，向该地址发请求即可触发；可复制，请妥善保存以免恶意调用
凭证校验系统生成的 Secret，用于 HMAC 签名校验；支持显示 / 复制
调用示例提供多语言请求示例（curl / python / node / go / java / rust）、HMAC 算法说明、响应体结构
高级配置IP 白名单（可选开关）

5.3 推送配置

字段可选值说明
推送渠道① 关联的 Agent ② Agent 绑定的 bot二选一或多选：通过 Agent 会话推送结果，或通过 Agent 绑定的机器人推送
推送对象部门 / 人 / 群可选择部门、指定人员或群组。若推送对象是群，机器人需已加入该群；请确保推送对象在机器人可用范围内

5.4 静默条件

创建任务时可填写静默条件：满足该条件时跳过本次推送（不打扰）。例如：

  • 「若群内今日无新增消息，则不给我发送消息」
  • 「若汇总结果为空 / 无异常项，则跳过本次推送」
💡 与「机器人主动发消息」工具协作
自动化任务负责触发 + 喂指令 + 决定推送，"消息送达"由 Agent / bot 完成。需保证：① Agent 已挂载所需工具；② 推送渠道选 bot 时 Agent 已绑定有效机器人；③ 推送对象在机器人可用范围内。

  • 下半部分：执行历史列表，按时间倒序，每行展示：执行时间、耗时、结果（成功/失败/已跳过）、错误原因（失败时）、查看执行日志

6.3 执行结论与记录标题

自动化任务每次执行都会生成一条执行结论（Agent 的产出结果）。该结论进入记录列表时，标题统一格式化为「自动化：任务名称」，便于与手动对话产生的记录区分。

失败重试与告警

  • 单次执行失败 → 自动重试 1 次（间隔 1 分钟）
  • 重试仍失败 → 记录到执行历史，标红显示，并通过 WrokHub Studio 站内消息告警给创建人
  • 连续 3 次执行失败 → 自动停用该任务，避免持续无效消耗
  • 命中静默条件 → 记录为「已跳过」，不计入失败
8典型场景串联
场景 1 · 定时触发 · 群运维日报
每晚 18:00 把当天运维情况发给负责人
① 手动创建：关联「群运维 Agent」/ 定时触发-每天 18:00 / 指令="请汇总今日 POPO 用户反馈群里的关键问题，整理成日报" / 推送渠道=Agent 绑定的 bot / 推送对象=@张振 / 静默条件="若群内今日无新增消息则不推送"
② 18:00 触发 → Agent 生成日报 → 校验静默 → bot 推给张振；记录列表出现「自动化：群运维日报」。
场景 2 · 事件触发 · 文档变更播报
指定云文档一变更，就总结并通知相关人
事件触发-选择「用户的云文档变更」→ 下级条件「指定云文档」选择目标文档；关联「文档助手」/ 指令="总结本次文档变更要点" / 推送对象=产品群。文档一变更即触发执行并推送。
场景 3 · WebHook 触发 · 发布播报
外部 CI 构建完成回调，触发发布播报
WebHook 触发-复制系统生成的地址与凭证，配置到外部 CI；构建完成后 CI POST 该地址 → 触发「发布播报 Agent」生成发布说明 → 推送到发布群。
场景 4 · 对话创建
C 端用户对话式创建自动化任务
用户对 Agent 说"帮我创建一个自动化任务" → Agent 回吐表单卡片（补充触发方式、执行动作、时间/频率、推送对象、静默条件）→ 用户填写提交 → 任务创建成功，创建人记为该用户。
9权限与依赖
  • 可见性与隔离：B 端租户 / C 端用户均可创建；数据按创建人所属租户 / 个人空间隔离，互不可见
  • 对 Agent 的依赖：任务关联的 Agent 被删除 → 任务自动停用并提示
  • 对推送的依赖：推送渠道选 bot 时，Agent 需已绑定有效机器人，且推送对象（人/群）在机器人可用范围内；推送对象是群时机器人需已加群
  • 对触发源的依赖：事件触发依赖对应事件源接入；WebHook 触发依赖外部系统按签名规范调用
  • 时区：定时触发默认采用创建人时区；一期不开放任务级时区设置
10交付与验收
  • 菜单出现在 AI 工厂下，B 端与 C 端用户均可进入、创建任务，列表正确记录创建人
  • 三种触发类型均可配置并端到端跑通：定时（每天 18:00 发日报）/ 事件（云文档变更触发）/ WebHook（外部回调触发）
  • 手动创建与对话创建（表单卡片）两种方式均可成功创建任务
  • 推送渠道（Agent / bot）与推送对象（部门 / 人 / 群）配置生效，消息正确送达
  • 静默条件命中时正确跳过推送，记录为「已跳过」
  • 执行生成执行结论，记录列表标题为「自动化：任务名称」
  • 任务停用 / 启用 / 编辑 / 删除均可正常生效；执行成功 / 失败 / 跳过均有完整历史可查
  • 失败重试、告警、连续失败自动停用机制按 7 节描述工作
  • Agent 被删除后，关联任务自动停用并在列表标记
